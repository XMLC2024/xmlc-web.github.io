    配置 pip 与 Python

    Note

    如果你无法确定你是否有pip,可以在控制终端中输入 pip,若显示不存在
    请先进行安装

    点击此处https://www.python.org/
    在Downloads中找到合适版本下载
    下载完成后请使用上述方法进行测试pip是否正常！

    下载 Mkdocs

    pip下载Mkdocs

    此处只介绍使用 pip 下载的方法
    打开控制终端，在控制终端中执行 

    pip install mkdocs

    docker pull squidfunk/mkdocs-material

    提示

    你可以使用mkdocs来测试是否安装成功

    Github Pages

    开始前请看

    请注意，这里只是简单介绍一下Github Pages
    如果你在之前从未了解过Github Pages
    请你现在网上稍微了解一下在进行

    Github存储库创建

    1. 首先，创建新的存储库，请将其命名为 你想要的名称+.github.io

    警告!

    你必须将创建的存储库公开！
    否则无法进行下面的步骤!

    1. 打开Github Desktop,如果没有，请点击https://desktop.github.com/进行安装
    2. 在Github Desktop中Clone你的存储库 File>Clone repository...
    3. Clone你的存储库到任意文件夹中，建议命名为site，并且是二级文件夹。
    实例

    D:/web/site

    命名为site并且放到二级文件夹的原因

    如果你将名称命名为site,在你以后build时
    将不再需要把生成后的文件拷贝到你clone的存储库内
    放到二级文件夹的原因:
    到时候你能看懂你在干什么。

    GithubPages

    在你的网页存储库中，点击Settings
    左侧栏目中选择Pages,找到Branch
    选择 main/root 并点击Save保存设置即可

    必看

    如果你在以后提交更改后仍然没有反应，请对下方情况进行排查
    - 提交分支错误
    - 没有启用Github Pages
    如错误仍没有解决，请参考可爱的百度
    或者可爱的谷歌

    到现在为止，基础配置已经结束，你已经可以开始了。
